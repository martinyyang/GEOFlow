# 快速上手

这页不是完整部署手册，而是一条尽量短、尽量稳的 GEOFlow 上手路径。

## 第一步：先确定目标

先明确三件事：

- 你要做什么类型的站点或内容系统
- 你希望服务谁
- 你准备先沉淀哪类知识内容

如果这一步不清楚，后面的模型、提示词和模板都容易跑偏。

## 第二步：先把系统跑起来

最推荐的方式是 Docker。

基本步骤：

1. 克隆仓库
2. 复制 `.env.example` 为 `.env`
3. 按需修改端口、站点地址和安全密钥
4. 启动 `web + postgres + scheduler + worker`

目标不是一次性配全，而是先拿到一个可访问的前台和后台。

## 第三步：登录后台

后台地址：

- `/geo_admin/`

默认管理员账号：

- 用户名：`admin`
- 密码：`admin888`

登录后第一件事：

- 修改管理员密码
- 修改 `APP_SECRET_KEY`

## 第四步：先配置模型

进入：

`AI 配置中心 -> AI 模型管理`

先接入一个稳定、响应较快的聊天模型。  
标题生成、正文生成、审核流程不需要一开始就用最复杂的模型组合。

## 第五步：准备基础素材

至少先准备这些：

- 一个标题库
- 一个知识库
- 一套正文提示词
- 一个作者
- 一个分类

如果没有真实知识库，建议先不要急着大规模创建任务。

## 第六步：创建第一个任务

任务最小组合建议：

- 标题库：已有可用标题
- 模型：一个稳定聊天模型
- 提示词：正文生成提示词
- 分类：明确的内容分类
- 审核方式：先走草稿 / 审核流程

第一批任务的目标是验证链路，而不是追求规模。

## 第七步：看第一篇内容的完整链路

你应该至少确认这几件事：

1. 任务能正常入队
2. Worker 能正常执行
3. 文章能生成到草稿
4. 审核页能看到内容
5. 发布后前台能正常展示

只要这五步打通，后面再做优化和扩展都更稳。

## 第八步：再做优化，不要反过来

建议顺序是：

1. 先知识库
2. 再模型和提示词
3. 再任务和审核
4. 再前台模板和主题
5. 最后再做 CLI / Skill / API 自动化

不要一开始就追求：

- 模板很复杂
- 自动化很多
- 批量任务很多

先把真实内容链路跑通，系统才有价值。
